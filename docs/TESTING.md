# אסטרטגיית בדיקות

## סוגי בדיקות

1. **בדיקות יחידה אוטומטיות (Vitest)** — כל הלוגיקה: מנוע, Solver, מחולל,
   אחסון, אימות. רצות בסביבת Node ללא DOM (localStorage מדומה בזיכרון).
2. **בדיקות ידניות** — ממשק המשתמש, נגישות, RTL (טבלה בהמשך).

## הרצה

```bash
npm test          # הרצה חד־פעמית
npm run test:watch  # מצב צפייה
```

לפני כל commit משמעותי מריצים גם:

```bash
npm run typecheck
npm run lint
npm run build
```

- כל הצבעים נכונים אך במקומות שגויים ✔
- כל הצבעים נכונים ובמקומות הנכונים ✔
- אין אף צבע נכון ✔
- רצף באורך משתנה (2, 4, 6) ✔
- מספר צבעים קטן מאורך הרצף ללא כפילויות (שגיאת אימות) ✔

## בדיקת גרסת ה־EXE

1. `npm run dist:win`
2. להריץ את `release/Super Mastermind <version>.exe` (portable) או להתקין עם ה־Setup.
3. לוודא: פתיחת חלון, משחק מלא, שמירת הגדרות אחרי סגירה ופתיחה מחדש,
   עבודה ללא רשת (לנתק רשת ולוודא שהכול פועל).
4. לוודא שכפתור „חשוף פתרון (DEV)” **אינו** קיים בגרסת Production.

## הוספת בדיקה חדשה

קובץ חדש תחת `src/tests/*.test.ts` נאסף אוטומטית (ראו `vite.config.ts`).
ללוגיקה עם אקראיות — להזריק `seededRng`. לאחסון — להשתמש ב־MemoryStorage
כמו ב־`storage.test.ts`.

## מגבלות כיסוי ידועות

- אין בדיקות רכיבי React אוטומטיות (jsdom/RTL) — ה־UI מכוסה ידנית בלבד.
- צלילים (WebAudio) ו־download/upload קבצים אינם מכוסים אוטומטית.
- electron/main.ts נבדק ידנית בלבד (הרצת האפליקציה).
