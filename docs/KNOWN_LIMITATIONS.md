# מגבלות ידועות וחובות טכניים

מסמך זה מוצג בגלוי ומתוחזק יחד עם הקוד. שום רכיב לא מוצג כגמור אם אינו כזה.

## 1. משחק פעיל אינו שורד סגירת אפליקציה

- **המגבלה:** סגירת האפליקציה באמצע משחק מאבדת את המשחק (הגדרות, סטטיסטיקות
  והיסטוריה כן נשמרות).
- **מתי מורגש:** סגירה בטעות באמצע משחק ארוך.
- **סיבה:** הרצף הסודי חי בזיכרון בלבד (ADR-003) — שמירתו לדיסק הייתה חושפת אותו.
- **פתרון עתידי:** סריאליזציה עם ערפול קל + שחזור בהפעלה, בהסכמה מפורשת של המשתמש.

## 2. במרחבים עצומים הניתוח מוערך ולא מדויק

- **המגבלה:** מעל 1,000,000 רצפים (למשל 16 צבעים × אורך 8 עם כפילויות) הספירה
  היא הערכה סטטיסטית; אי אפשר להוכיח שם סתירה, פתרון יחיד או עובדות מיקום.
- **מתי מורגש:** רק בהגדרות קיצוניות; בכל התבניות המובנות הניתוח מדויק.
- **טיפול קיים:** מוצג במפורש „בהערכה כ־N” ו„רמת ודאות: מוערכת” (ADR-005).
- **פתרון עתידי:** מנייה אינקרמנטלית עם עצירה מוקדמת, או סינון הדרגתי של הקבוצה.

## 3. יצירת חידת „ניחוש מכריע” קשה אורכת עד ~3 שניות

- **מתי מורגש:** דרגות קשה/מומחה עם דרישת פתרון יחיד.
- **טיפול קיים:** רץ ב־Web Worker עם חיווי „בונה חידה…” — הממשק אינו קופא.
- **פתרון עתידי:** הקטנת מספר קריאות `computePossibleSet` במחולל (שימוש חוזר
  בקבוצת הפתרונות במקום חישוב מחדש לכל מועמד).

## 4. אין בדיקות אוטומטיות לרכיבי React

- **המגבלה:** ה־UI מכוסה בבדיקות ידניות בלבד (מתועדות ב־TESTING.md).
- **סיכון:** רגרסיה ויזואלית/אינטראקטיבית לא תיתפס אוטומטית.
- **פתרון עתידי:** הוספת @testing-library/react + jsdom למסכים המרכזיים.

## 5. חבילות ההתקנה אינן חתומות

- **המגבלה:** Windows SmartScreen עשוי להציג אזהרה בהרצת ה־EXE/מתקין.
- **סיבה:** אין אישור Code Signing (כרוך בעלות).
- **עקיפה:** "More info" → "Run anyway". הקבצים נבנים מקומית מהמקור.

## 6. מחיקת „נתוני אתר” של האפליקציה מוחקת את השמירות

- **המגבלה:** הנתונים ב־localStorage של פרופיל Electron (ADR-001).
- **עקיפה קיימת:** ייצוא/ייבוא JSON של חידות ומשחקים.
- **פתרון עתידי:** אחסון בקבצים דרך IPC (שינוי נקודתי ב־storage/local.ts).

## 7. ביטול חישוב ב־Worker אינו ממומש

- **המגבלה:** בקשה שהתייתרה (למשל ניחוש חדש לפני שהניתוח הקודם הסתיים) ממשיכה
  לרוץ ברקע; התוצאה פשוט נזרקת (דגל stale).
- **השפעה:** בזבוז CPU קל במצבים גדולים; אין השפעה על נכונות.
- **פתרון עתידי:** פרוטוקול ביטול (terminate/restart או בדיקת דגל בתוך הלולאה).

## 8. קיצורי מקלדת לצבעים מוגבלים ל־10 הראשונים

- **המגבלה:** מקשים 1–9 ו־0 בלבד; צבעים מעבר לעשירי נבחרים בעכבר בלבד.
- **השפעה:** רק בפלטות של 11+ צבעים.
